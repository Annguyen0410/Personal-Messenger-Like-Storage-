Single-page Electron renderer split across four files. `monitor.html` is the CSP-restricted shell (`default-src 'self'`, whitelisted USGS/Carto/unpkg/eonet/opensky/wheretheiss.at domains) declaring every widget container as `<div class="widget" data-widget="...">`. `monitor-preload.js` is the only bridge to the main process, exposing exactly two methods via `contextBridge` — `fetchQuakes` (invokes `monitor:quakes`) and `close` (sends `monitor:close`) — mounted on `window.monitorApi`. `monitor.css` provides the dark HUD theme; `monitor.js` is a self-contained IIFE bootstrapped in strict mode that owns all UI logic: a `WIDGETS` registry drives per-widget visibility persisted in `localStorage` under `wm_widget_<id>`, a document-level delegated click handler manages `.widget-toggle` buttons, and a sidebar tab system switches between RISK / ROUTE / SCENARIO / FINANCE / INFRA panels. Data sources are embedded mock datasets (`MOCK`, `NEWS`, `THREATS`, `COUNTRIES`, plus finance/crypto/commodities tables) rendered immediately, then polled every 60 s via `api.fetchQuakes()` through the IPC bridge, falling back to `MOCK.features` on failure. Leaflet 1.9.4 (loaded from unpkg) renders quakes as `L.circleMarker`s on a Carto Dark basemap; scenario routes are drawn on a separate `scenarioLayer`. There are no external module dependencies beyond Leaflet.
