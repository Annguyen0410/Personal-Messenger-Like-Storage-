The root is the Electron application shell wired across three cooperating layers with no shared package between them — communication happens exclusively through `ipcMain.handle` / `contextBridge`:
- `main.js` owns the primary `BrowserWindow`, enforces single-instance via `app.requestSingleInstanceLock`, registers the custom `local-file://` streaming scheme so the renderer never touches the filesystem, and persists `messages.json`/`settings.json` under `app.getPath('userData')`. It also spawns a second fullscreen BrowserWindow (`monitorWin`) for the World Monitor dashboard on demand.
- `preload.js` exposes exactly one surface (`window.api`) to the sandboxed renderer, mapping each `<domain>:<action>` IPC channel (store, file, voice, notification, theme) to a promise-returning handler; it additionally exposes `window.monitorApi` for the monitor window's quake fetch/close calls.
- `renderer.js` + `index.html` + `styles.css` are pure browser-side UI; `renderer.js` keeps all state in-memory and calls `api.save()` after every mutation. `app.js` is a legacy duplicate kept alongside it for compatibility.
Build/deploy is delegated to `electron-builder` configured in `package.json` targeting Windows NSIS + portable, with icons under `assets/buildResources`.
